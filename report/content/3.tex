\subsection{Descarga de datos y modelos}

Para garantizar la reproducibilidad y facilitar el despliegue del proyecto, se ha implementado un sistema automatizado de gestión de recursos siguiendo principios de diseño \texttt{SOLID} y \texttt{Clean Code}. Este sistema se encarga de la descarga y organización tanto del dataset como de los modelos de lenguaje.

\subsection{Tokenización}

La tokenización es el proceso de convertir el texto plano en secuencias de números (\textit{tokens}) que el modelo puede procesar. Dado que cada arquitectura (Gemma y Qwen) utiliza su propio vocabulario, se ha implementado un servicio desacoplado que encapsula la lógica de los tokenizadores. Para maximizar el rendimiento en hardware Apple Silicon, el proceso se ejecuta utilizando el backend \texttt{MPS} (Metal Performance Shaders), moviendo los tensores resultantes directamente a la GPU del dispositivo.

\begin{itemize}
    \item \textbf{Gemma:} Utiliza un tokenizador basado en \textit{Byte-Pair Encoding} (BPE).
    \item \textbf{Qwen:} Utiliza un esquema optimizado para múltiples lenguajes.
\end{itemize}

\subsection{Truncado}

Para manejar las limitaciones de memoria y la longitud máxima de contexto de los modelos, se aplica una estrategia de truncado. Las secuencias que exceden la longitud máxima definida (512 tokens) se cortan para asegurar que los tensores resultantes tengan dimensiones compatibles con la arquitectura del Transformer seleccionado.

\subsection{Padding}

Dado que los modelos de Deep Learning requieren procesar datos en lotes (\textit{batches}) de tamaño uniforme, se aplica \textit{padding} a las secuencias más cortas. Se añaden tokens especiales hasta alcanzar la longitud máxima predefinida, permitiendo la creación de tensores rectangulares eficientes. El sistema guarda estos tensores en formato \texttt{.pt} dentro de la carpeta \texttt{.data/preprocessed/}, evitando repetir el proceso en ejecuciones futuras.

\subsection{Manejo Secuencias Largas}

Para el manejo de secuencias extremadamente largas en diálogos complejos, se han considerado estrategias como el truncado inteligente, priorizando los turnos más recientes de la conversación para preservar el contexto inmediato del diálogo.
