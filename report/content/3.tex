\section{Preprocesado}
\subsection{Descarga de datos y modelos}

Para garantizar la reproducibilidad y facilitar el despliegue del proyecto, se ha implementado un sistema automatizado de gestión de recursos siguiendo principios de diseño \texttt{SOLID}.

\subsubsection{Autenticación y Seguridad (.env)}

Debido a que modelos de última generación requieren autenticación para su descarga, se ha integrado la librería \texttt{python-dotenv}. El sistema carga automáticamente un \texttt{HF\_TOKEN} desde un archivo \texttt{.env}, permitiendo que las librerías de Hugging Face gestionen las descargas de forma transparente.

\subsubsection{Arquitectura del Sistema de Ingestión}

\begin{itemize}
    \item \textbf{Gestión de Modelos (\texttt{.models/}):} Se centraliza la descarga de GGUFs y Tokenizers.
    \item \textbf{Estructuración de Diálogos (JSON Chat Format):} Se transforman los archivos de SGD a un formato intermedio de mensajes (\textit{role/content}). El sistema evita conversiones redundantes mediante chequeos de integridad.
\end{itemize}

\subsubsection{Gestión del Almacenamiento y Caché (Hugging Face Hub)}

Para optimizar el uso del disco y evitar descargas redundantes de archivos de gran tamaño (9GB-10GB por modelo), el sistema utiliza el almacenamiento centralizado de \texttt{huggingface\_hub}. En el entorno de desarrollo (macOS), los modelos en formato \textit{safetensors} se gestionan automáticamente en la ruta del sistema:
\begin{center}
    \texttt{\textasciitilde/.cache/huggingface/hub}
\end{center}
Esta arquitectura permite que el entrenamiento detecte instantáneamente si un recurso ya está disponible localmente, realizando únicamente una comprobación de integridad (\textit{ETags}) antes de proceder, lo que reduce drásticamente el tiempo de preparación en ejecuciones sucesivas.

\subsection{Tokenización y Chat Templates}

Se ha implementado el uso de \textbf{Strict Chat Templates}. El sistema utiliza el método \texttt{apply\_chat\_template} para asegurar que el modelo recibe los tokens de control exactos (\texttt{<|im\_start|>}, \texttt{<|user|>}, etc.).

\subsubsection{Adaptación a Versiones Modernas de Transformers}

Durante la ejecución, se ha observado y corregido el uso de parámetros obsoletos siguiendo las advertencias del sistema. Un ejemplo relevante es la transición de \texttt{torch\_dtype} hacia el nuevo estándar \texttt{dtype} en los cargadores de modelos.

\subsection{Truncado y Padding}

Se aplica una estrategia de truncado a \textbf{512 tokens} y \textit{padding} dinámico. Esto permite la creación de tensores rectangulares eficientes para el entrenamiento, optimizando el uso de la memoria unificada del sistema.

\subsection{Persistencia de Tensores}

El sistema guarda los resultados como archivos de tensores de PyTorch (\texttt{.pt}) dentro de \texttt{.data/preprocessed/}, organizados jerárquicamente por modelo y partición (\texttt{train} y \texttt{test}). 

\subsection{Manejo de Secuencias Largas}

Para diálogos extensos, se ha implementado una estrategia de \textbf{ventana deslizante}. El sistema prioriza los turnos más recientes del diálogo, garantizando que el modelo mantenga la información relevante para generar la respuesta actual sin exceder su ventana de contexto nativa.
