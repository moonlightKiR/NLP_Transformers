\section{Selección del Caso de Uso y Dataset}

\subsection{Caso de Uso: Generación de Respuestas en Diálogo}
Para esta práctica, se ha seleccionado la tarea de \textbf{Generación de Respuestas en Diálogo}. Esta tarea consiste en desarrollar modelos capaces de procesar el historial de una conversación y generar una respuesta coherente, relevante y gramaticalmente correcta.

\subsection{Dataset Seleccionado: SGD (Schema-Guided Dialogue)}
Se ha optado por utilizar el dataset \textbf{SGD (Schema-Guided Dialogue)} de Google. La elección se fundamenta en que representa el estado del arte actual para modelos multidominio, ofreciendo más de 16,000 diálogos en 20 dominios distintos.

\subsection{Modelos Seleccionados}
Para la comparativa, se han seleccionado dos arquitecturas de vanguardia que ofrecen un equilibrio óptimo entre tamaño y capacidad de razonamiento en hardware local:

\begin{itemize}
    \item \textbf{Qwen 3.5-2B (Alibaba Qwen):} Un modelo de 2 billones de parámetros altamente optimizado. Se ha seleccionado esta versión frente a la de 4B para garantizar la estabilidad del entrenamiento en sistemas con 18GB de RAM unificada, manteniendo un rendimiento sobresaliente.
    \begin{itemize}
        \item \textit{Source:} \href{https://huggingface.co/Qwen/Qwen3.5-2B}{Qwen/Qwen3.5-2B}
        \item \textit{GGUF:} \texttt{Qwen3.5-2B-Q4\_K\_M.gguf} (Unsloth)
    \end{itemize}
    
    \item \textbf{Phi-4-mini Instruct (Microsoft):} Modelo de 3.8 billones de parámetros diseñado específicamente para el razonamiento compacto. Es una de las arquitecturas más eficientes del mercado, optimizada para seguir instrucciones complejas.
    \begin{itemize}
        \item \textit{Source:} \href{https://huggingface.co/microsoft/Phi-4-mini-instruct}{microsoft/Phi-4-mini-instruct}
        \item \textit{GGUF:} \texttt{Phi-4-mini-instruct-Q4\_K\_M.gguf} (Unsloth)
    \end{itemize}
\end{itemize}

\subsection{Evolución de la Selección y Desafíos Técnicos}
La selección final es fruto de un proceso de iteración donde se descartaron otros modelos por limitaciones de acceso y arquitectura:
\begin{itemize}
    \item \textbf{Llama 3.2 3B:} Se intentó integrar esta arquitectura, pero al ser un repositorio \textit{Gated}, requiere validación manual y el uso de un token de Hugging Face.
    \item \textbf{Gemma 4 E2B-it:} Se descartó tras detectar una incompatibilidad estructural entre los pesos (\textit{safetensors}) y la implementación de la clase \texttt{Gemma} en la librería MLX.
    \item \textbf{Qwen 3.5 4B:} Se intentó su uso, pero el consumo de memoria Metal superó los límites del hardware durante el proceso de cálculo de gradientes.
\end{itemize}
